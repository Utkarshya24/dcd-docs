## Dockerfile (for all)
Use this Dockerfile for Gin, Echo, or Fiber apps:

```dockerfile
FROM golang:1.22-alpine AS build
WORKDIR /app
COPY . .
RUN go mod tidy
RUN go build -o app .

FROM alpine:latest
WORKDIR /app
COPY --from=build /app/app .
EXPOSE 8080
CMD ["./app"]
```

## Best Practices
- Bind to 0.0.0.0 (not localhost).
- Use environment variables for DB, secrets, keys.
- Optimize Docker image size with multi-stage builds.
- Enable middleware (logging, recovery, compression).
- Monitor metrics (CPU, RAM) for high-performance Go apps.

## Troubleshooting
| Problem            | Cause                           | Solution                                 |
| ------------------ | ------------------------------- | ---------------------------------------- |
| App unreachable    | Bound to `localhost` only       | Use `0.0.0.0:$PORT`                      |
| Build failed       | Missing `go.mod` / dependencies | Run `go mod tidy` before deploy          |
| Crash after deploy | Missing env vars                | Configure env vars in DCDeploy dashboard |
| Large Docker image | Using single-stage build        | Switch to multi-stage Docker build       |
| Port mismatch      | Wrong exposed/listen port       | Ensure `EXPOSE 8080` and use `$PORT` env |
